# Knowledge standards

- Write for a maintainer who knows Python but not the current implementation.
- Lead with purpose and boundaries.
- Use stable names matching the source tree and configuration.
- Keep procedures short and verification-oriented.
- Never include secrets, personal data, raw transcripts, or recordings.
- Avoid source line numbers and facts likely to become stale without a trigger.
- Link to the canonical contract for exact defaults, keys, and command syntax.
